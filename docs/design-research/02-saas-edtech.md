# Реестр паттернов: лидеры SaaS/edtech 2025–2026

> Исследование 2026-06-07: 12 живых сайтов препарированы (Linear, Stripe, Vercel, Framer,
> Raycast, Dia, Notion, Brilliant, Maven, Untools, Skyeng, Практикум, Coursera, Khan Academy)
> + ~15 teardown-статей. Полные источники в конце.

## A. Структура главной

1. **Linear — нумерованный конвейер.** 5 секций-глав (1.0 Intake → 5.0 Monitor), каждая = экран:
   заголовок-глагол + абзац + живой скриншот. → TerenLabs: нумерованный путь предпринимателя
   1.0 Академия → 2.0 Тесты → 3.0 Кейсы → 4.0 Обзоры → 5.0 Финмодель.
2. **Stripe — доверие 2-м экраном.** Сразу под hero — логотипы/доказательства, до фич.
   → У нас вместо логотипов: проверяемость источников (БНС, НК-2026).
3. **Raycast — секция чистой эмоции** без единой фичи. → «Защита от потерь, не успешный успех»
   заслуживает отдельного экрана-манифеста.
4. **Notion — заголовок-крючок** («Meet the night shift») + расшифровка сабтайтлом.
   → Перекликается с регистром крючков Адиля.
5. **Brilliant — двойной CTA по аудиториям** (learner/teacher) + продают МЕХАНИКУ обучения,
   не каталог. Авторитет создателей (MIT/Harvard), не клиентов.
6. **Khan Academy — бесплатность как миссия.** «Почему бесплатно» проговаривается на главной,
   иначе бесплатность читается как дешёвка. КРИТИЧНО для TerenLabs.
7. **Dia — сценарии дня** вместо фич («Start your day two steps ahead»).
   → «Перед подписанием аренды — открой обзор ниши».
8. **Skyeng — некруглые числа** («893 627 учеников») убедительнее круглых.
9. **Консенсус-каркас (8 секций):** Hero (5–10 слов + 2 CTA) → продукт в движении →
   ≤3 метрики над сгибом → проблема→решение → 3–4 исхода → соц-доказательство → прайсинг → CTA.

## B. Карточки

10. **Maven:** без цены/рейтинга на карточке — название, длительность, лицо эксперта.
11. **Coursera:** карточка отвечает «что это и от кого», детали внутри.
12. **Практикум:** ровно 3 оси меты (уровень / длительность / статус-бейдж).
13. **Untools:** однострочник-выгода начинается с глагола («Identify root causes…»).
14. **Иллюстрации vs фото:** софт → реальный UI; live-обучение → лица; self-paced → фирменные
    иллюстрации. TerenLabs: уроки/тесты — low-poly арты, финмодели/обзоры — реальные скриншоты.

## C. Прайсинг и CTA

15. Каноника: 3 тира, «Most popular» в центре (+38%), названия по исходу, annual-дефолт.
16. **CTA-цифры:** первое лицо «Start MY trial» +90% CTR; 1 CTA на страницу = 13.5% vs 10.5% у трёх.
    Глаголы: Get/Start/Try/Unlock. → «Посчитать мой бизнес», не «Подробнее».
17. **Бесплатный тир легитимизирует платные:** страница цен = сначала «Бесплатно навсегда»
    как полноценный тир, потом FinModel $20 / BizPlan $20.
18. Двойной CTA в hero: главный = действие, вторичный = посмотреть/поговорить.

## D. Показ продукта

19. **Скриншот реального UI в hero** (+32% регистраций vs стоковая иллюстрация). Тренд: micro-demo 10–15 с.
20. **Linear:** каждая секция = затемнённый под тему фрагмент UI с подсветкой нужной зоны.
21. **Brilliant: продукт и есть демо** — интерактивные задачки до регистрации.
    → СИЛЬНЕЙШИЙ паттерн: живой вопрос теста прямо на лендинге.
22. **Stripe:** рандомная задержка в анимации набора (не равномерные 100мс) — «человечность» деталей.
23. **Dia:** в мокапах узнаваемый контент. → У нас «Кофейня в Актобе», не «Бизнес 1».

## E. Соц-доказательства без фейка

24. Считаемые факты вместо мнений: динамические счётчики контента (не зашитые).
25. Авторитет метода: «по НК КЗ 2026», «только данные БНС» = проверяемость как доверие.
26. Прозрачность методологии («Как мы считаем» с 5 наставлениями) бьёт generic-отзывы.
27. Качество самого сайта = главный сигнал доверия (школа Linear/Vercel).

## F. Типографика

28. H1: clamp(32px, 8vw, 72px); Linear: 62px/800/lh1.16. Body ≥16px, контент-платформы 18px.
    H1 lh 1.1–1.2, body 1.5–1.8.
29. Колонка лонгрида: 50–75 символов (≈600–720px при 18px).
30. ≤3 семейства. Vercel: mono для цифр = «инженерная честность» → JetBrains Mono для всех
    чисел финмоделей.

## G. Тёмные темы

31. Лидеры уходят от ХОЛОДНЫХ синеватых серых к тёплым (Linear redesign 2026).
    Наш навы — синий: компенсировать тёплыми ступенями между навы и текстом.
32. **Elevation = светлота, не тени:** base + 3 ступени по +5–8% luminance, ≤4–5 уровней,
    AA-контраст проверять на каждом.
33. **Dark-first:** проектировать на тёмном холсте, светлая тема — оверрайд.
34. «Linear style» (глоу+градиенты под блюром) — уже клише: 1 глоу на hero, не везде.
35. **Blueprint grid Vercel:** едва видимая сетка (opacity 5–20%, шаг 16–24px) за контентом
    расчётных секций — «чувствуешь раньше, чем видишь».

## Приоритеты для TerenLabs

1. Структура главной: Linear-нумерация пути + Khan-блок «почему бесплатно» + Stripe-доверие 2-м экраном.
2. **Живой вопрос теста на лендинге** (Brilliant) — демо + вовлечение без регистрации.
3. Соц-доказательство: счётчики из данных + «Как мы считаем» + источники.
4. Карточки: 3 оси меты + однострочник-глагол.
5. Тёмная тема: 4 ступени поверхностей с тёплым сдвигом, бирюза только акцент, mono-цифры,
   blueprint-grid в расчётных секциях.
6. CTA: первое лицо + исход, 1 главный на экран.
